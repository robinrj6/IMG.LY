# Header

The Header component is used across all IMG.LY websites.

<Story of={HeaderStories.Default} />

<br />

This component has two states and will break at the `lg` [breakpoint](?path=/docs/design-system-breakpoints--page).

## Import

```tsx
import { Header } from "@imgly-labs/imgly-website-components";
```

or

```html
<div id="imgly-website-components-header"></div>
```

## Props

| Name                   | Type    | Default                | Description                                  |
| ---------------------- | ------- | ---------------------- | -------------------------------------------- |
| `hiring`/`data-hiring` | boolean | defined in `config.ts` | Display the `Remote works! Join us.` banner. |

```tsx
<Header hiring />
```

or

```html
<header id="imgly-website-components-header" data-hiring="true"></header>
```

## Usage

```tsx
<Header />
```

> The `Header` will need a top margin of `2rem` for the `sm` [breakpoint](?path=/docs/design-system-breakpoints--page) if `HeaderCareers` is used so that the component can be positioned correctly.
> Switch to a custom implementation if this is not possible.

## Customization

The `Header` is comprised of the following elements:

```tsx
<HeaderWrapper>
  <HeaderLogo />
  <HeaderContainer>
    <HeaderChildren />
  </HeaderContainer>
  <HeaderCareers />
  <HeaderMenu>
    <HeaderMenuChildren />
  </HeaderMenu>
</HeaderWrapper>
```

The `HeaderContainer` will be shown on resolutions greater than `lg` and the `HeaderMenu` on smaller resolutions.

> The relative element will need a top margin of `2rem` for the `sm` breakpoint.
